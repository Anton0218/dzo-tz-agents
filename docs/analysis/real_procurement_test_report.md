## 3. Методология

### 3.1 Независимый экспертный анализ
Каждый документ был проверен по чек-листу, соответствующему:
- **Агент ТЗ**: 8 обязательных разделов (ALGORITHM.md агента 2)
- **Агент ДЗО**: 6 обязательных полей (ALGORITHM.md агента 1)

### 3.2 Тестирование через API
Документы поданы через REST API (`POST /api/v1/process/{agent}`) с вложением
в виде base64-текстового файла. Результаты получены из `GET /api/v1/jobs/{id}`.

### 3.3 Уровни тестов

| Уровень | Маркер pytest | Требует LLM | Описание |
|---|---|---|---|
| Pipeline | _(без маркера)_ | ❌ | Проверяет прохождение документа через API (received → extract → prepare_input) |
| Rules-engine | _(без маркера)_ | ❌ | Проверяет наличие/отсутствие ключевых разделов в тексте документа |
| E2E | `@pytest.mark.e2e` | ✅ | Проверяет полное LLM-решение агента |

## 6. Ключевые выводы

### ✅ Что работает хорошо
- Полный API-пайплайн: `received → extract_attachments_done → prepare_input` — стабильно
- Критические флаги («НЕ ПРИЛОЖЕНА», «отсутствует гарантия») — корректно обнаруживаются
- Структура агентов соответствует реальным корпоративным чек-листам 44-ФЗ/223-ФЗ

### ⚠️ Известные ограничения (rules-engine без LLM)
- **Ложноположительные срабатывания** при keyword-matching (rbank case)
- Keyword `г.` в технических характеристиках воспринимается как «г. [город]» → место поставки
- Rules-engine не различает структуру документа — только присутствие слов

### 🔧 Рекомендации
1. **Подключить LLM**: `OPENAI_API_KEY=sk-...` или `LLM_BACKEND=github_models` + `GITHUB_TOKEN`
2. **Добавить структурный парсер**: определять раздел по заголовку, а не по содержимому
3. **Расширить E2E тесты**: добавить документы с неоднозначными кейсами (частично заполненные разделы)

## 7. Запуск тестов

```bash
# Pipeline + Rules-engine (без LLM, ~5 сек)
pytest tests/test_real_procurement_docs.py -v

# Только rules-engine (без API)
pytest tests/test_real_procurement_docs.py::TestRealDocumentRulesEngine -v

# E2E с реальным LLM
LLM_BACKEND=openai OPENAI_API_KEY=sk-... pytest tests/test_real_procurement_docs.py -m e2e -v

# Все тесты проекта
pytest tests/ -v --tb=short
```
